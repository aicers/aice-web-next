# 탐지

탐지 페이지는 사이드바에서 접근합니다. 백엔드에서 생성된 탐지 결과를
필터링하고 검토하며 개별 항목을 자세히 살펴보는 허브입니다.

페이지 조회에는 `detection:read` 권한이 필요합니다. 기본 제공 역할인
Security Monitor, Tenant Administrator, System Administrator는 이
권한을 기본으로 부여받습니다. `detection:read`를 부여한 사용자 정의
역할도 접근할 수 있습니다.

![탐지 페이지](../assets/detection-ko.svg)

> 위 그림은 와이어프레임 대체본입니다. 이 페이지는 작성 환경에서
> 접근할 수 없는 REview 백엔드에 의존하므로, 스테이징 환경이
> 준비되면 실제 스크린샷으로 교체합니다.

## 레이아웃

페이지는 네 개의 영역으로 구성됩니다. 결과 리스트가 작업
공간의 중심이며, 보조 영역은 결과에 집중할 수 있도록 간결하게
유지됩니다.

### 추천 / 저장된 필터 레일

왼쪽의 슬림 레일은 두 섹션으로 구성됩니다.

- **추천 필터** — 선별된 시작점.
- **저장된 필터** — 사용자가 직접 저장한 필터.

좁은 뷰포트에서는 레일이 아이콘만 표시되도록 축소되고, 데스크톱
너비에서는 섹션 제목이 함께 나타납니다.

### 상단 바

메인 영역 상단에는 **필터** 버튼과 **활성 필터 칩 바**가 있습니다.
**필터** 버튼을 클릭하면 오른쪽에 필터 드로어가 열리고, 그 옆의 칩
바는 현재 탭에 적용된 필터를 요약해 보여 줍니다.

### 결과 리스트

결과 리스트는 화면 대부분을 차지하며 탐지 이벤트를 한 줄씩
렌더링합니다. 페이지에 진입하면 기본 필터(**최근 1시간**)로
쿼리가 자동 실행되어 결과 영역이 비어 있는 상태로 남지 않습니다.

각 항목은 두 줄로 구성됩니다.

- 첫째 줄 — 심각도 배지, 시각, 종류 / 공격 유형, MITRE 카테고리,
  신뢰도(소수점 두 자리), 트리아지 요약(`정책 수 · 최대 점수`).
- 둘째 줄 — `출발지 IP:포트 (국가) → 목적지 IP:포트 (국가)` 와
  센서 이름. `MultiHostPortScan`·`ExternalDdos` 같이 주소 필드가
  배열인 이벤트는 각 쪽의 첫 항목에 **`+N건`** 버튼을 덧붙여
  표시합니다. 이 버튼을 클릭하면 접힌 주소 전체 목록이 팝오버로
  열리며, 포트 배열을 전달하는 스캔 이벤트(`PortScan`)에서는 접힌
  포트 목록이 표시됩니다. 리스트에서 벗어나지 않고도 숨겨진 값을
  확인할 수 있습니다. 이벤트의 전체 필드는 조사 뷰에서 확인합니다.

#### 헤더

리스트 위쪽에는 건수와 범위가
`탐지된 이벤트 <범위> / <전체건수>` 형태로 렌더링됩니다 — 예를
들어 1 284건짜리 결과의 첫 페이지는 `탐지된 이벤트 1-50 / 1,284`
로 표시됩니다. 이와 함께 **마지막 갱신** 표시, **새로 고침**
버튼, **CSV 다운로드** 버튼이 놓입니다. CSV 다운로드는 이번
단계에서 비활성화되어 있습니다 — 내보내기는 이후 단계에서
연결됩니다.

#### 행 상호작용

- 행 본문을 클릭하면 간단 미리보기 인스펙터가 열립니다. 넓은
  뷰포트(1280px 이상)에서는 결과 리스트와 공간을 나누는 오른쪽
  인라인 패널로 도킹되어 리스트 너비가 비례해서 줄어듭니다. 좁은
  뷰포트에서는 리스트 너비를 유지하도록 오버레이 드로어로 열립니다.
  어느 형태든 선택한 이벤트의 요약 헤더(심각도, 시각, 종류, 센서)와
  전체 조사 뷰로 이동하는 버튼을 보여 줍니다. 상세 패널은 이후
  단계에서 연결되며, 그때까지는 나머지 영역에 "곧 추가됩니다"
  안내가 표시됩니다.
- 행 오른쪽의 **조사 열기** 버튼은 전체 조사 뷰로 바로 이동합니다.
  출발지 / 목적지 주소가 없는 호스트 기반 이벤트(예: `ExtraThreat`,
  `WindowsThreat`)는 로케이터를 만들 수 없으므로, 해당 행에서는
  아무 동작도 하지 않는 버튼을 남기지 않고 버튼 자체를 숨깁니다.

#### 반응형 전략

리스트는 열을 감추는 대신 밀도를 조정합니다. 뷰포트가 좁아지면
간격이 좁아지고 출발지 / 목적지 쌍이 세로로 쌓입니다. 심각도,
시각, 종류, 주소 정보는 모든 지원 너비에서 항상 표시됩니다.

### 비어 있음 / 로딩 / 오류 상태

- 쿼리 결과가 0건이면 "일치하는 결과 없음" 상태가 표시되며
  필터 범위를 넓히거나 칩을 제거하라는 안내가 함께 나타납니다.
- 쿼리가 진행 중일 때는 헤더의 새로 고침 아이콘이 회전하며 이전
  결과는 유지됩니다.
- 쿼리가 실패하면 본문이 오류 메시지로 교체되며, 새로 고침
  버튼으로 재시도할 수 있습니다.

### 분석 스트립

결과 아래에는 현재 결과 집합의 집계 정보를 위한 분석 스트립이
마련되어 있습니다. 기본적으로 접혀 있으며, 확장하면 이번
단계에서는 비어 있는 플레이스홀더 패널이 나타납니다.

## 활성 필터 칩 바

칩 바는 드로어가 닫혀 있어도 모든 적용 필터를 항상 표시합니다.

- **단일 값 필드**(기간, 출발지, 목적지, 신뢰도 하한/상한)는
  값을 그대로 표시합니다. 예: `출발지: 10.0.0.5`.
- **3개 이하의 배열 필드**는 항목마다 칩이 하나씩 렌더링됩니다.
- **그보다 큰 배열**은 하나의 집계 칩으로 압축되며 개수가 함께
  표시됩니다. 예: `호스트명: 7개 선택됨`. 칩 본문을 클릭하면
  포함된 값 목록이 인라인 팝오버로 열려, 드로어를 거치지 않고도
  접힌 값들을 확인할 수 있습니다. `×` 버튼은 해당 필드 전체를
  제거합니다.
- 어느 칩이든 `×`를 누르면 독립 커밋입니다. 해당 필드(또는 개별
  항목)가 즉시 제거되고 쿼리가 재실행됩니다. 드로어의 **적용**
  버튼은 여러 필드 편집을 일괄 커밋할 때 쓰며, 단일 항목 제거에는
  사용하지 않습니다.
- **기간** 또는 **범위** 칩의 본문을 클릭하면 필터 드로어가 열려
  시간 범위를 조정할 수 있습니다. 이 단계에서는 드로어가 해당
  컨트롤만 제공합니다.
- 전용 드로어 컨트롤이 아직 없는 필드(출발지, 목적지, 종류,
  호스트명 등)의 칩 본문을 클릭하면 **값 팝오버**가 대신 열립니다.
  팝오버는 커밋된 값을 표시하고, 전용 드로어 컨트롤이 이후 단계에서
  추가된다는 안내와 함께 **필터에서 제거** 버튼을 제공하여 칩의
  `×` 외에도 필터에서 해당 필드를 제거할 수 있도록 합니다. 이렇게
  하여 지금 단계에서도 모든 칩 본문이 의미 있는 동작을 제공하며,
  드로어가 편집할 수 없는 필드를 편집할 수 있는 것처럼 오인하지
  않도록 합니다. 이후 단계에서 해당 필드의 드로어 컨트롤이
  추가되면 칩 본문은 자동으로 해당 필드에 포커스한 드로어를 여는
  방식으로 전환됩니다.

요약 로직은 `@/lib/detection/summarize-filter` 에 있는 순수 헬퍼로,
칩을 렌더링하는 모든 곳에서 재사용되므로 표시 문구와 제거 시
동작이 항상 일치합니다.

> **전방 호환성 — 쿼리 모드.** `Filter` 타입에는 향후 검색 언어
> UI를 위한 `mode: "query"` 분기가 예약되어 있습니다. v1 에서는
> `mode: "structured"` 만 구현되어 있으며, 쿼리 분기에 대해서는
> 칩을 필드 단위로 분해하지 않고 이후 단계에서 편집 가능한 단일
> 알약(pill) 형태로 렌더링합니다. 배경은 엄브렐라 이슈의 전방
> 호환성 섹션을 참고하세요.

## 필터 드로어

필터 드로어는 조회할 탐지 이벤트의 시간 창을 지정하는 곳입니다.
상단 바의 **필터** 버튼으로 열면 오른쪽에서 슬라이드되어
나타납니다.

![탐지 필터 드로어](../assets/detection-drawer-ko.png)

### 기간

**기간** 섹션은 흔히 쓰는 상대 시간 창을 칩으로 제공합니다:
`최근 1시간`, `최근 12시간`, `최근 1일`, `최근 1주`, `최근 1개월`,
`최근 3개월`, `최근 6개월`, `최근 1년`, `최근 3년`. 칩을 선택하면
해당 창의 시작과 끝이 **시간 범위** 입력에 채워집니다.

### 시간 범위

두 개의 `datetime-local` 입력으로 시작과 종료 시각을 명시적으로
지정할 수 있습니다. 어느 한쪽이라도 편집하면 기간 칩 선택이
해제됩니다. 사용자가 직접 편집한 범위는 더 이상 퀵 셀렉트
창이 아니기 때문입니다.

### 적용

**적용**을 클릭하거나 드로어에 포커스가 있는 상태에서 `Enter` 키를
누르면 현재 드래프트가 활성 탭의 필터로 커밋되고 쿼리가 실행됩니다.
적용 후에는 드로어가 닫힙니다. 적용 없이 드로어를 닫으면(닫기
버튼 또는 `Esc`) 편집 중이던 내용은 유지되며 다음에 드로어를 열
때 다시 나타납니다.

종료 시각이 시작 시각보다 이르거나 같은 범위는 허용되지 않으며
인라인 유효성 검사 메시지가 표시됩니다.

### 필터 저장

**필터 저장** 버튼은 적용 버튼 옆에 표시되지만 이번 단계에서는
비활성화되어 있습니다. 필터 이름 지정 플로우는 이후 탐지
단계에서 연결됩니다.
